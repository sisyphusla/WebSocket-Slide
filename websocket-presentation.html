Connected Users

The Cost of WebSocket

01

Server Resources

1 user = 1 persistent connection.
10K users = 10K open sockets.
HTTP serves and moves on — WebSocket doesn't.

02

Scaling is Harder

Stateful — client is tied to one server.
Adding more servers doesn't automatically help.

03

More Things Can Go Wrong

Network drops, proxy timeouts, mobile sleep.
You must handle all of them.

04

Infrastructure Cost

Load balancers, firewalls, proxies — all need long-lived connection support.

What We Learned Today

1 HTTP is request-response. It wasn't designed for real-time.
2 WebSocket gives us a persistent, two-way connection with very low overhead.
3 A connection has a lifecycle — open it, keep it alive, handle disconnections gracefully.
4 Choose the right message pattern — broadcast, room, or one-to-one.
